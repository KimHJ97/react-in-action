# 페이지 캐싱

## 1. 풀 라우트 캐시

풀 라우트 캐시는 Next.js App Router에서 빌드 타임 또는 첫 요청 시 렌더링된 페이지의 HTML과 React Server Components Payload(RSC Payload)를 캐싱하여 재사용하는 메커니즘입니다.

 - 페이지 단위 전체 결과(HTML)를 캐싱해서, 페이지 렌더링 없이 빠르게 서빙

<br/>

### 1-1. 풀 라우트 캐시 적용 대상 (Static Page)

기본적으로 정적 렌더링이 적용되는 모든 라우트에 활성화된다.

 - 동적 함수 미사용: cookies(), headers(), searchParams 같은 동적 함수를 사용하지 않아야 한다.
 - 동적 데이터 없음: fetch 요청에 revalidate: 0 또는 cache: 'no-store' 같은 동적 옵션이 설정되지 않아야 한다.
 - 동적 경로가 정적: 동적 라우트라도 generateStaticParams를 사용해 빌드 시 모든 가능한 경로를 생성하면 정적 렌더링이 적용된다.
