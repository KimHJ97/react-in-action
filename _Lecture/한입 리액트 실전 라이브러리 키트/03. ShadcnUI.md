# ShadcnUI

## 1. ShadcnUI 소개

- Bootstrap, Material UI처럼 현대 웹 개발에 필요한 대부분의 필수 UI 요소를 제공하는 디자인 라이브러리이다
- 2024년 State of Frontend 설문에서 머테리얼 UI와 부트스트랩을 제치고 디자인 시스템 부문 1등을 차지했다
- TailwindCSS 기반으로 제작되어 커스텀이 편리하고 호환성 문제가 적다

### 1-1. ShadcnUI 설치

- 공식 사이트: https://ui.shadcn.com/

```bash
npm install -D @types/node
npx shadcn@latest init
```

- **tsconfig.json**
  - 경로 별칭 추가

```json
{
  "files": [],
  "references": [
    { "path": "./tsconfig.app.json" },
    { "path": "./tsconfig.node.json" }
  ],
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

- **tsconfig.app.json**

```json
{
  "compilerOptions": {
    // ..
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

- **vite.config.ts**

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";
import path from "path";

export default defineConfig({
  plugins: [react(), tailwindcss()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
});
```

#### 설치된 파일

- **components.json 파일**
  - 셋업 완료 후 생성되는 Shadcn UI의 설정 파일이다
  - 컴포넌트 설치 시 어떤 방식으로, 어디에 설치할지 결정하는 기준이 된다
  - `aliases` 옵션으로 UI 컴포넌트의 설치 경로(`src/components/ui`)가 지정되어 있다

#### 사용할 컴포넌트 추가 및 사용하기

```bash
npx shadcn@latest add button
npx shadcn@latest add input
npx shadcn@latest add textarea
npx shadcn@latest add sonner
```

```tsx
import "./App.css";
import { Button } from "./components/ui/button";
import { cn } from "./lib/utils";

function App() {
  const isActive = true;

  return (
    <>
      <div>
        <Button>버튼</Button>

        <div
          className={cn(
            "w-10 text-sm",
            isActive ? "text-green-500" : "text-red-500",
          )}
        >
          isActive
        </div>
        <div className={isActive ? "text-green-500" : "text-red-500"}>
          isActive
        </div>

        <div className="text-primary">Primary</div>
      </div>
    </>
  );
}

export default App;
```

### 1-2. cn 헬퍼 함수

- `src/lib/utils.ts`에 자동 생성되는 TailwindCSS 클래스 도우미 함수이다
- 여러 개의 클래스 이름을 인자로 받아 충돌 없이 안정적으로 적용될 수 있는 클래스 문자열로 변환한다
- 조건에 따라 클래스를 유동적으로 설정해야 할 때 유용하다 (예: `cn(isActive ? "text-green-500" : "text-red-500")`)
- 콤마로 여러 클래스를 전달하는 등 다양한 기능을 제공한다

## 2. Shadcn/UI 핵심 컴포넌트

### 2-1. 버튼

- `variant` prop으로 버튼의 스타일 유형을 조절한다
- 제공되는 variant: `default`, `secondary`, `destructive`, `outline`, `ghost`, `link`
- `destructive`는 빨간색의 부정적/파괴적 액션용 버튼, ghost는 호버 시 배경이 나타나는 버튼, link는 링크처럼 밑줄이 생기는 버튼 등이다
- 색상은 `index.css`의 CSS 변수에 미리 설정된 테마 값을 기반으로 적용된다
- variant 값을 외울 필요 없이, 이런 방식으로 설정 가능하다는 점만 알면 된다

```javascript
function App() {
  return (
    <div>
      <Button>버튼</Button>
      <Button variant={"destructive"}>버튼</Button>
      <Button variant={"ghost"}>버튼</Button>
      <Button variant={"link"}>버튼</Button>
      <Button variant={"outline"}>버튼</Button>
      <Button variant={"secondary"}>버튼</Button>
    </div>
  );
}
```

### 2-2. Input & Textarea

- `npx shadcn@latest add input`으로 설치하며, `src/components/ui/input.tsx`로 생성된다
- HTML `<input>` 태그와 사용법이 완전히 동일하다
- placeholder, onChange 등 기존 HTML 속성을 그대로 사용한다

```javascript
function App() {
  return (
    <div className="p-5">
      <Input />
      <Input placeholder="Enter text" />
      <Textarea id="textarea-message" placeholder="Type your message here." />
    </div>
  );
}
```

### 2-3. Sonner (토스트 메시지)

- 최상위 부모 레이아웃의 Toaster 컴포넌트를 추가한다.
- 이후 토스트 메시지가 필요한 곳에서 `toast()` 함수를 사용한다.

```javascript
function App() {
  return (
    <div className="p-5">
      <Toaster></Toaster>

      <Button
        variant="outline"
        onClick={() => {
          toast("토스트 메시지", {
            position: "top-center",
          });
        }}
      >
        버튼
      </Button>
    </div>
  );
}
```

### 2-4. Carousel

- `Carousel` > `CarouselContent` > `CarouselItem` 구조로 배치하며, 이를 컴파운드 컴포넌트 패턴(합성 컴포넌트 패턴)이라고 부른다
- 한 번에 여러 아이템을 표시하려면 CarouselItem에 `className="basis-1/3"` 등으로 flex-basis 값을 설정한다
- `CarouselPrevious`와 `CarouselNext` 컴포넌트로 좌우 이동 버튼을 추가할 수 있다
- 버튼이 보이지 않을 경우 Carousel에 마진(`m-10` 등)을 적용하여 여백을 확보한다

```javascript
<Carousel>
  <CarouselContent>
    <CarouselItem></CarouselItem>
    <CarouselItem></CarouselItem>
  </CarouselContent>
</Carousel>
```

### 2-5. Popover

- `Popover` > `PopoverTrigger` + `PopoverContent` 구조로 사용한다
- PopoverTrigger는 팝오버 창을 여는 버튼 역할, PopoverContent는 실제 팝오버 창 역할을 한다
- 중요: 반드시 `@/components/ui`에서 import해야 하며, `radix-ui`에서 직접 import하면 디자인과 편의기능이 빠진 채로 동작한다
- `asChild` prop을 PopoverTrigger에 추가하면 자식으로 전달된 Button 컴포넌트를 그대로 트리거로 사용할 수 있다

```javascript
<Popover>
  <PopoverTrigger>Open</PopoverTrigger>
  <PopoverContent>Content!</PopoverContent>
</Popover>
```

### 2-6. Dialog

- `Dialog` > `DialogTrigger` + `DialogContent` 구조이며, `DialogContent` 내부에 `DialogHeader` > `DialogTitle` + `DialogDescription`을 배치한다
- 창 바깥을 클릭하면 다이얼로그가 닫힌다
- Popover와 마찬가지로 `radix-ui`가 아닌 `@/components/ui`에서 import해야 한다

```javascript
<Dialog>
  <DialogTrigger>Open Dialog</DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Title</DialogTitle>
      <DialogDescription>description ...</DialogDescription>
    </DialogHeader>
    <div>body ...</div>
  </DialogContent>
</Dialog>
```

```javascript
      </AlertDialogDescription>
    </AlertDialogHeader>
    <AlertDialogFooter>
      <AlertDialogCancel onClick={() => console.log("cancel")}>
        Cancel
      </AlertDialogCancel>
      <AlertDialogAction onClick={() => console.log("continue")}>
        Continue
      </AlertDialogAction>
    </AlertDialogFooter>
  </AlertDialogContent>
</AlertDialog>
```
