# 서버 상태관리와 Tanstack Query

## 1. 서버 상태 관리

### 1-1. 상태 분류

- `지역 상태`: 특정 컴포넌트에서만 접근 가능한 상태 (예: Input 입력값)
- `전역 상태`: 전체 컴포넌트에서 접근 가능한 상태 (예: 테마, 세션 정보)
- `서버 상태`: API 요청과 관련된 데이터로, 별도의 유형으로 구분하여 관리한다

### 1-2. 서버 상태를 별도로 관리해야 하는 이유

- API 요청에는 isLoading, status, error, 캐시 옵션 등 관리해야 할 데이터가 매우 많다
- Zustand 같은 전역 상태 도구로 관리하려면 각 요청마다 수많은 State와 액션 함수를 만들어야 한다
- 대부분의 서비스가 API를 여러 개 호출하므로, 복잡한 코드를 각 요청마다 일일이 작성해야 한다
- 지역 상태로 관리해도 여러 컴포넌트에서의 데이터 공유가 어렵고 코드 복잡도는 동일하다

TaskStack Query를 이용하면 한 줄의 코드로 API 요청, 데이터, 로딩/에러 상태를 관리할 수 있으며, 요청 실패시 자동 재요청, 중복 요청 방지 등 다양한 편의 기능을 제공한다.

## 3. 데이터 조회 요청 관리하기

### 3-1. Tanstack Query 초기 설정

- `라이브러리 설치`

```bash
npm i @tanstack/react-query
```

- `main.tsx`
  - 프로젝트의 QueryClient 적용
  - QueryClient는 서버 상태(캐시 데이터)를 저장하는 역할을 한다

```tsx
import { QueryClientProvider, QueryClient } from "@tanstack/react-query";

const queryClient = new QueryClient();

createRoot(document.getElementById("root")!).render(
  <BrowserRouter>
    <QueryClientProvider client={queryClient}>
      <App />
    </QueryClientProvider>
  </BrowserRouter>,
);
```

### 3-2. useQuery 훅 사용법

- `기존 코드`

```javascript
async function fetchTodos() {
  const response = await fetch(`${API_URL}/todos`);
  if (!response.ok) throw new Error("Fetch Failed");

  const data: Todo[] = await response.json();
  return data;
}

export default function TodoListPage() {
  const [todos, setTodos] = useState<Todo[]>([]);
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState();

  const fetchData = async () => {
    setIsLoading(true);
    try {
      const data = await fetchTodos();
      setTodos(data);
    } catch (error) {
      setError(error as any);
    } finally {
      setIsLoading(false);
    }
  };

  useEffect(() => {
    fetchData();
  }, []);

  return (
    <div className="flex flex-col gap-5 p-5">
      <h1 className="text-2xl font-bold">TodoList</h1>
      <TodoEditor />
      {todos.map((todo) => (
        <TodoItem key={todo.id} {...todo} />
      ))}
    </div>
  );
}
```

- `useQuery 사용`

```javascript
async function fetchTodos() {
  const response = await fetch(`${API_URL}/todos`);
  if (!response.ok) throw new Error("Fetch Failed");

  const data: Todo[] = await response.json();
  return data;
}

export default function TodoListPage() {
  const {
    data: todos,
    isLoading,
    isError,
  } = useQuery({
    queryFn: fetchTodos,
    queryKey: ["todos"],
    retry: 0, // 재시도 횟수 (기본 3)
  });

  if (isError) return <div>오류가 발생했습니다</div>;
  if (isLoading) return <div>로딩중입니다</div>;

  return (
    <div className="flex flex-col gap-5 p-5">
      <h1 className="text-2xl font-bold">TodoList</h1>
      <TodoEditor />
      {todos?.map((todo) => (
        <TodoItem key={todo.id} {...todo} />
      ))}
    </div>
  );
}
```

### 3-3. 코드 분리 패턴

- `코드 분리 패턴`
  - API 호출 함수(예: `fetchTodos`)를 별도 파일(예: `api/todo.ts`)로 분리한다
  - 커스텀 훅(예: `useTodosData`)을 만들어 useQuery 호출을 캡슐화한다
  - 컴포넌트에서는 커스텀 훅만 호출하여 데이터를 사용한다

```javascript
// api/todo.ts
export async function fetchTodos() {
  const response = await fetch(`${API_URL}/todos`);
  if (!response.ok) throw new Error("Fetch Failed");

  const data: Todo[] = await response.json();
  return data;
}

// hooks/quries/use-todos.data.ts
export function useTodosData() {
  return useQuery({
    queryFn: fetchTodos,
    queryKey: ["todos"],
  });
}

// TodoListPage
export default function TodoListPage() {
  const { data: todos, isLoading, isError } = useTodosData();

  if (isError) return <div>오류가 발생했습니다</div>;
  if (isLoading) return <div>로딩중입니다</div>;

  return (
    <div className="flex flex-col gap-5 p-5">
      <h1 className="text-2xl font-bold">TodoList</h1>
      <TodoEditor />
      {todos?.map((todo) => (
        <TodoItem key={todo.id} {...todo} />
      ))}
    </div>
  );
}
```
